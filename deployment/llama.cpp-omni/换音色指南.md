# llama.cpp-omni 换音色完整指南

## 原理概述

llama.cpp-omni 的语音合成音色由一段**参考音频**决定。音色在两个层面起作用：

1. **LLM 层**：参考音频被编码为 APM embedding 注入 system prompt，让 LLM 知道要模仿什么音色来生成语音 token
2. **Vocoder 层（C++ Token2Wav）**：`prompt_cache.gguf` 里存着从参考音频预计算出的 speaker embedding 和流式缓存，Token2Wav 据此把语音 token 合成为目标音色的波形

换音色就是**两边都换成同一个参考音频**的产物——缺一不可，不然 LLM 认为的音色和合成出来的音色对不上。

## 换音色完整步骤

假设你有一个新的参考音频文件（比如 `new_voice.m4a`），以下用 `<变量名>` 表示需要替换为实际路径的占位符。

### 第 1 步：准备参考音频

要求：WAV 格式，16kHz 单声道，3–10 秒清晰无噪音的单人语音。

```bash
# 如果是 m4a/mp3 等格式，先转成 wav（16kHz 单声道）
ffmpeg -y -i new_voice.m4a -ar 16000 -ac 1 new_voice_full.wav

# 截取其中清晰的 6 秒片段（调整 -ss 起始时间）
ffmpeg -y -i new_voice_full.wav -ss 10 -t 6 new_voice_6s.wav
```

### 第 2 步：用 Python 从参考音频提取 prompt_bundle 文件

将以下脚本保存为 `extract_prompt_bundle.py`（仓库中没有此文件，需自行创建）：

```python
#!/usr/bin/env python3
"""
从参考音频提取 prompt_bundle 三个文件（spk、tokens、mel），
供 C++ Token2Wav 生成 prompt_cache.gguf。

用法:
    python extract_prompt_bundle.py \
        --ref-audio new_voice_6s.wav \
        --model-dir <PYTHON_MODEL_DIR> \
        --output-dir tools/omni/assets/default_ref_audio
"""

import argparse
import os
import numpy as np
import torch
import torchaudio
import torchaudio.compliance.kaldi as kaldi
import s3tokenizer
import onnxruntime
from librosa.filters import mel as librosa_mel_fn


# ==================== mel 频谱计算 ====================

_mel_basis = {}
_hann_window = {}

def mel_spectrogram(y, n_fft=1920, num_mels=80, sampling_rate=24000,
                    hop_size=480, win_size=1920, fmin=0, fmax=8000):
    """计算 mel 频谱（与 CosyVoice2 / Step-Audio2 对齐）"""
    global _mel_basis, _hann_window
    key_mel = f"{fmax}_{y.device}"
    key_win = str(y.device)
    if key_mel not in _mel_basis:
        mel = librosa_mel_fn(sr=sampling_rate, n_fft=n_fft, n_mels=num_mels, fmin=fmin, fmax=fmax)
        _mel_basis[key_mel] = torch.from_numpy(mel).float().to(y.device)
        _hann_window[key_win] = torch.hann_window(win_size).to(y.device)
    y = torch.nn.functional.pad(
        y.unsqueeze(1),
        (int((n_fft - hop_size) / 2), int((n_fft - hop_size) / 2)),
        mode="reflect",
    ).squeeze(1)
    spec = torch.stft(
        y, n_fft, hop_length=hop_size, win_length=win_size,
        window=_hann_window[key_win], center=False,
        pad_mode="reflect", normalized=False, onesided=True, return_complex=True,
    )
    spec = torch.sqrt(torch.view_as_real(spec).pow(2).sum(-1) + 1e-9)
    spec = torch.matmul(_mel_basis[key_mel], spec)
    return torch.log(torch.clamp(spec, min=1e-5))


# ==================== 主流程 ====================

def extract(ref_audio: str, model_dir: str, output_dir: str, device: str = "cuda"):
    # ---------- 1. 提取 speech tokens ----------
    audio_16k = s3tokenizer.load_audio(ref_audio, sr=16000)
    mels_s3 = s3tokenizer.log_mel_spectrogram(audio_16k)
    mels_s3, mels_s3_lens = s3tokenizer.padding([mels_s3])
    tokenizer = s3tokenizer.load_model(
        os.path.join(model_dir, "speech_tokenizer_v2_25hz.onnx")
    ).to(device).eval()
    tokens, tokens_lens = tokenizer.quantize(
        mels_s3.to(device), mels_s3_lens.to(device)
    )

    # ---------- 2. 提取说话人 embedding ----------
    spk_feat = kaldi.fbank(audio_16k.unsqueeze(0), num_mel_bins=80, dither=0, sample_frequency=16000)
    spk_feat = spk_feat - spk_feat.mean(dim=0, keepdim=True)
    option = onnxruntime.SessionOptions()
    option.graph_optimization_level = onnxruntime.GraphOptimizationLevel.ORT_ENABLE_ALL
    option.intra_op_num_threads = 1
    spk_session = onnxruntime.InferenceSession(
        os.path.join(model_dir, "campplus.onnx"),
        sess_options=option, providers=["CPUExecutionProvider"],
    )
    spk_emb = torch.tensor(
        spk_session.run(None, {spk_session.get_inputs()[0].name: spk_feat.unsqueeze(0).cpu().numpy()})[0],
        device=device,
    )

    # ---------- 3. 提取 mel 频谱 ----------
    audio_raw, sr = torchaudio.load(ref_audio, backend="soundfile")
    audio_raw = audio_raw.mean(dim=0, keepdim=True)
    if sr != 24000:
        audio_raw = torchaudio.transforms.Resample(orig_freq=sr, new_freq=24000)(audio_raw)
    prompt_mel = mel_spectrogram(audio_raw).transpose(1, 2).squeeze(0)
    prompt_mels = prompt_mel.unsqueeze(0).to(device)

    UP_RATE = 2
    prompt_mels = torch.nn.functional.pad(
        prompt_mels,
        (0, 0, 0, tokens.shape[1] * UP_RATE - prompt_mels.shape[1]),
        mode="replicate",
    )

    # ---------- 4. 按 C++ 要求截断 mel 并保存 ----------
    T = tokens.shape[1]
    PRE_LOOKAHEAD = 3
    expect_T_mel = (T - PRE_LOOKAHEAD) * UP_RATE   # C++ 要求

    os.makedirs(output_dir, exist_ok=True)
    spk_emb.squeeze().cpu().float().numpy().tofile(os.path.join(output_dir, "spk_f32.bin"))
    tokens.squeeze().cpu().int().numpy().tofile(os.path.join(output_dir, "prompt_tokens_i32.bin"))
    prompt_mels[:, :expect_T_mel, :].squeeze(0).cpu().float().numpy() \
        .flatten().tofile(os.path.join(output_dir, "prompt_mel_btc_f32.bin"))

    print(f"Done! T_token={T}, T_mel={expect_T_mel}")
    print(f"Files saved to {output_dir}/")


if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="从参考音频提取 prompt_bundle 文件")
    parser.add_argument("--ref-audio", "-r", required=True, help="参考音频 WAV 路径")
    parser.add_argument("--model-dir", "-m", required=True,
                        help="包含 speech_tokenizer_v2_25hz.onnx 和 campplus.onnx 的目录")
    parser.add_argument("--output-dir", "-o", required=True,
                        help="输出目录（通常是 tools/omni/assets/default_ref_audio）")
    parser.add_argument("--device", "-d", default="cuda", help="cuda 或 cpu")
    args = parser.parse_args()
    extract(args.ref_audio, args.model_dir, args.output_dir, args.device)
```

运行：

```bash
python extract_prompt_bundle.py \
    --ref-audio new_voice_6s.wav \
    --model-dir <PYTHON_MODEL_DIR> \
    --output-dir tools/omni/assets/default_ref_audio
```

同时把参考音频也复制过去：

```bash
cp new_voice_6s.wav tools/omni/assets/default_ref_audio/default_ref_audio.wav
```

执行完成后 `tools/omni/assets/default_ref_audio/` 目录下应有：

| 文件 | 内容 | 说明 |
|------|------|------|
| `default_ref_audio.wav` | 参考音频原文件 | 用于 LLM system prompt 和 Python Token2Wav |
| `spk_f32.bin` | 192 维说话人 embedding | float32，768 字节 |
| `prompt_tokens_i32.bin` | 参考音频的 speech token 序列 | int32 |
| `prompt_mel_btc_f32.bin` | 参考音频的 mel 频谱图 | float32，长度 `(T_token - 3) * 2` |

### 第 3 步：移除旧的 prompt_cache.gguf，设置导出目录

```bash
GGUF_DIR=<MODEL_DIR>/token2wav-gguf

# 备份旧 cache
mv "$GGUF_DIR/prompt_cache.gguf" "$GGUF_DIR/prompt_cache.gguf.old"

# 准备导出目录
mkdir -p /tmp/t2w_export
rm -f /tmp/t2w_export/prompt_cache.gguf
```

### 第 4 步：启动服务，让 C++ 自动生成新的 prompt_cache.gguf

启动时通过环境变量 `T2W_EXPORT_CACHE_DIR` 告诉 C++ 把新生成的 cache 导出到指定目录。
由于 `prompt_cache.gguf` 已被移除，C++ 会自动 fallback 到第 2 步生成的 prompt_bundle 文件进行实时计算，并将结果导出。

```bash
cd <LLAMACPP_ROOT> && \
REF_AUDIO=<REF_AUDIO_WAV> \
MODEL_DIR=<MODEL_DIR> \
PYTHON_CMD=<PYTHON_CMD> \
LLAMACPP_ROOT=<LLAMACPP_ROOT> \
CPP_MODE=simplex \
T2W_EXPORT_CACHE_DIR=/tmp/t2w_export \
bash WebRTC_Demo/oneclick.sh start
```

环境变量说明：

| 变量 | 作用 | 示例 |
|------|------|------|
| `REF_AUDIO` | 参考音频路径，传给 LLM system prompt 做音色 embedding | `/path/to/new_voice_6s.wav` |
| `MODEL_DIR` | GGUF 模型总目录 | `/path/to/MiniCPM-o-4_5-gguf` |
| `PYTHON_CMD` | Python 解释器路径 | `/path/to/python` |
| `LLAMACPP_ROOT` | llama.cpp-omni 仓库根目录 | `/path/to/llama.cpp-omni` |
| `CPP_MODE` | `simplex`（单工）或 `duplex`（双工） | `simplex` |
| `T2W_EXPORT_CACHE_DIR` | C++ 首次运行时将 prompt cache 导出到此目录 | `/tmp/t2w_export` |

启动后控制台会打印类似日志：

```
Token2Wav: prompt_cache failed, fallback to prompt_bundle from tools/omni/assets/default_ref_audio
[Token2Mel] exported prompt_cache.gguf: /tmp/t2w_export/prompt_cache.gguf
```

### 第 5 步：把导出的 prompt_cache.gguf 复制回模型目录

```bash
cp /tmp/t2w_export/prompt_cache.gguf "$GGUF_DIR/prompt_cache.gguf"
```

之后再启动就不需要移除旧文件和设置 `T2W_EXPORT_CACHE_DIR` 了，直接启动即可：

```bash
cd <LLAMACPP_ROOT> && \
REF_AUDIO=<REF_AUDIO_WAV> \
MODEL_DIR=<MODEL_DIR> \
PYTHON_CMD=<PYTHON_CMD> \
LLAMACPP_ROOT=<LLAMACPP_ROOT> \

```

## 要点总结

- 音色由**两层**决定：`REF_AUDIO`（LLM 层音色 embedding）+ `prompt_cache.gguf`（Vocoder 层音色缓存），**两个都要换且必须来自同一段参考音频**
- `prompt_cache.gguf` **必须由 C++ 代码自己生成**（通过 prompt_bundle fallback + `T2W_EXPORT_CACHE_DIR` 环境变量导出）
- 中间桥梁是 **prompt_bundle 三个文件**（`spk_f32.bin`、`prompt_tokens_i32.bin`、`prompt_mel_btc_f32.bin`），由上述 `extract_prompt_bundle.py` 脚本从参考音频提取
